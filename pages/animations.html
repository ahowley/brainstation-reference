CSS Animations

See this MDN page for an exaustive list of animatable CSS properties.

Transitions

  • We use transitions to smoothly ease between two values when a property changes in CSS.
  • The simplest way is to just provide a duration value to the transition property, which causes any animatable property to transition when changing value.
  • The transition property also accepts specified properties, rather than the default all: for example, transition: color 0.5s. Other properties will still transition immediately.
  • When specifying duration, we can provide a second value representing a delay. transition: color 0.5s 1s. There are other accepted values in transition: see here for examples of some others.
  • transition is a shorthand for a few other properties we can also set separately:
    • transition-duration
    • transition-property
    • transition-delay
    • transition-timing-function
  • When transitioning using longhand syntax, it's important to order animated properties carefully!
  • In order to transition multiple properties, we can separate each property in the transition with a comma in between. transition: color 0.1s 1s, background-color: 1s

Easing Functions

  • In order to vary how fast we transition as the transition occurs, we can use easing functions.
    • linear easing is the default, and happens steadily
    • ease-in accelerates into the animation
    • ease-in-out accelerates, then decelerates
    • ease-out decelerates
    • cubic-bezier(n,n,n,n) allows you to define a custom easing bezier curve. See cubic-bezier.com to visualize and build a bezier curve!
  • To add an easing function to a transition, we set the transition-timing-function property.

Non-Animatable Properties

  • display
  • background-image
  • border-style
  • position
  • float
  • font-family

CSS Transforms

  • CSS Transforms allow us to alter the visual position and size of an element.
    • transform: translate() scale() rotate() skew() - multiple transform functions can be provided with no separator.
  • rotate() rotates an element clockwise by a given number of degrees, # of turns, or radians (negative values can be used for counter-clockwise rotation).
  • skew() slants an element by a given number of degrees, # of turns, or radians.
  • scale() will increase or decrease the size of an element. It accepts either a single value, representing scaling on both axes, or two values, scaling x and y separately. It can be a number (multiplier) or a % value.
  • translate() will change an elements position. It accepts either a single value, translating on the x axis, or two values, translating on both axes.
  • Each of these has single-axis alternatives, like translateX() and translateY().
  • transform-origin allows us to set a new origin for the element.

@Keyframes

  • Keyframes can define more complex animations with precise control. For a detailed overview, see the MDN page here.
  • Can be applied to different element states, or page states like page-load (among others)
  • Keyframes can be looped and reversed
  • Allows for pure-CSS implementations of things like sprite-sheet animation and svg animation.
  • A simple keyframe definition would use @keyframe animation-name { from { property: value; } to { property: value; } }
  • To apply a keyframe animation to an element, we set the animation property. animation: animation-name
  • In place of "from" and "to", we can create more granular transition states using percentage values between 0 and 100, and define properties and values within each state.
  • We can use the animation-iteration-count propery to define how many times an animation should play, or we can set it to infinite.
  • We can use the animation-direction process to define whether an animation should play forward or backward.
  • We can use the animation-duration property to define a duration for an animation, in seconds.
  • To apply easing, we set the animation-timing-function property. There are many other configurable properties, which you can reference on MDN here.
