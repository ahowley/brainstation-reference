Classes & Object Oriented Programming

Classes give us a way to define a new "template" of sorts for objects. Almost everything in JavaScript is an object, and almost all of those things are instances of built-in classes.

Classes & Objects

  • Objects are just things that have properties that can be accessed. This means that Arrays are also objects, for example.
  • Understanding that an object can contain just about any other data type, including functions, is very important - when an object or class has a property that is also a function, it is also known as a method.
  • To define a new class, with starting parameters, we use the following syntax. The constructor method has to be named constructor, and nothing else - it's how we tell the code what to run when we instantiate a new object. 
    class MyClass {
  constructor(param1, param2) {
    this.prop1 = param1;
    this.prop2 = param2;
  }

  addPropertyExampleMethod(paramKey, paramValue) {
    this.paramKey = paramValue;
  }
}
  • The this keyword will return the object at the root of the current scope. For classes, it allows us to represent an object that is instantiated based on the class we're creating.
  • The simplest example of this is in the constructor above - we use this.prop1 = param1 to say "hey, whenever an object is made with this class, give it a property/key called prop1 with the value that's in param1".
  • To instantiate (create) a new object using a class as a template, we call its constructor method. To do this, we use the new keyword. 
    const myObject = new MyClass("prop1 value", "prop2 value");
myObject.addPropertyExampleMethod("prop3", "prop3 value");
  • The above example will create a new object with three properties - prop1, prop2, and prop3 - with values "prop1 value", "prop2 value", and "prop3 value".
  • Specifically, creating a class creates a new prototype for instantiated objects to reference. So if an object property doesn't exist, but does exist on its class/prototype, then when we try to access the property from the object, JS goes and finds the property on the parent, instead.
  • Classes can also contain data and properties that are not set when an object is instantiated, but instead, it exists in a constant state "publicly". An example of this would be the Math class - we don't need to create a new Math object to access its methods, like Math.random() or Math.max().
  • We use the static keyword in front of a method or property to define a new static method/property.
  • We can add new properties and methods to a prototype from anywhere using any object's prototype. Doing this to built-ins is generally not good practice, but knowing it's possible can help understand how prototypes work.
  • For example, the following example would add a "sum" method to all arrays that returns the sum of all values in the array. 
    Array.prototype.sum = function sum() {
  return this.reduce((acc, current) => acc + current, 0);
}
[1, 1].sum(); // 2

Object-Oriented Programming

  • Object-Oriented Programming is a way to capture similarities between different objects and give them a structure to draw from using Classes and Inheritance.
  • This allows us to structure code in a way that is easy to understand for humans, built on familiar concepts.
  • Inheritance provides us a way to create classes based on other classes. For example, we could have a parent class Person {}, then two different children classes: class Student extends Person {} and class Teacher extends Person {}.
  • The extends keyword is how we create a new class that inherits all the properties and methods from a parent class. This encourages reusing common code for similar classes by giving each of them a common prototype to extend.
