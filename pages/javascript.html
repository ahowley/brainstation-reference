Intro to JavaScript

JavaScript is a language that came out in the mid-1990s, and by coincidence was the first programming language that was adopted for the web. It's a high-level, interpreted language - meaning that it is very distant from "the metal," and is interpreted into computer instructions via an interpreter.

High-Level Info & Notes

  • JavaScript is one of the 3 core technologies that comprise any website or web application, along with CSS and HTML.
  • Because it's the "language of the web," its staying power is beyond strictly its objective appeal or supported features.
  • JavaScript can also be run on the server side, in the form of Node.js, which was built to allow JavaScript to be a full-stack language.

  • JavaScript files are denoted with the extension *.js.
  • To interact with the console in a browser, we refer to console - to log something to the console, for example, we can use console.log(...values)
  • JavaScript is generally agnostic to whitespace - it is not sensitive to whitespace in almost all situations, so we format our code according to style guides, convention, and preference.

Programming Terminology

  • An expression is kind of like a "sentence fragment" - it's smaller than a statement, and is a small chunk of code that returns a specific value.
  • A statement is the smallest standalone piece of code that "says something" - it's often (but not always) a single line of code, and in JavaScript, is separated from other statements by a semicolon.
  • Syntax is comprised of the rules and symbols that allow a computer to "read" the code you write - it's how you structure your statements and expression, and doesn't necessarily always make 100% sense - it's just something about a language you'll need to memorize to make code run.
  • A comment is text or content inside of a code file that isn't code, and isn't interpreted or compiled by the machine. These are typically there for DX (developer experience) and (hopefully) a client won't actually see them.
  • Immediately after the browser loads JavaScript, it will execute. This means that, all else being equal, JavaScript will execute from top to bottom, and can't "see" code that's below it (with some exceptions).
  • JavaScript is a loosely-typed language - as opposed to more strongly-typed languages. The Number data type is a good example of this.

Primitive Data Types

  • JavaScript has 5 immutable, primitive data types:
    • Number
    • String
    • Boolean
    • Null
    • Undefined
  • A Number is... a number. JavaScript (largely) doesn't distinguish between floating-point and integer numbers - instead, all numeric operations are typically done with these capital-N Numbers.
  • A String is a collection of ordered characters - basically, text data.
  • A Boolean is precisely either true or false - a boolean can't be anything other than these two.

Operators

  • An Operator is a "thing that does a thing." This would include things like the arithmetic operators: + - * / % ^
  • + - * / work as you might expect them to, and do not require whitespace around them to function.
  • ** is the exponent operator, and raises a number to the power of another number. 5**2 would return 25.
  • The % operator is known as the "modulo" operator, and it finds the remainer when dividing two numbers. For example, 9 % 5 would return 4, because 9 / 5 is 1, with a remainder of 4.
  • We can represent parenthetical operations the exact same way we would in pen-and-paper mathematics - wrapping expressions in parenthesis. (5 + 2) * 2 would return 7 * 2 = 14.
  • The bang operator, sometimes called the "not" operator, is represented with an exclamation mark - !
    • The bang operator can go in front of anything, and will interpret what follows as a boolean - specifically, the opposite of the boolean interpretation of what follows.
    • If that's confusing - the simplest example would be that !true would return false.
    • You can chain bang operators - for example, !!value would convert anything inside of value to a simple boolean representation.
    • Most data types have some state in which they're considered "falsy" - meaning they are calculated as false in boolean operations. You can reference falsy JS values on MDN here.
  • The and operator is represented with two consecutive ampersands - &&
  • The or operator is represented with two consecutive pipe characters - ||
