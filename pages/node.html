Node

Node is an interpreter built in C++ that allows us to run JavaScript in the terminal and create full-stack applications with one programming language. Learn more about node on node's website here.

Overview

  • Node is a runtime environment for JavaScript that exists outside of the browser. It allows us to build software locally, which enables backend development using JavaScript.
  • Modern Node is built on Google's open-source V8 JavaScript engine, which is a JIT-compiled interpreter built in C++.
  • To ease developing backend software using Node, there are many, many backend development frameworks available. The most popular at the time of writing is Express, which is a lightweight backend library for fast-paced development.
  • Like many other interpreted languages, Node provides a REPL (Read-Eval-Print-Loop) for JavaScript. To run this in our terminal with node installed, we can simply type node.
  • We can also code scripts and command line tools using Node JS that do helpful things to make our lives easier or automate repetitive tasks. To run a script, we can use node path/to/script.js. If we want to "watch" the file we're running and re-run it whenever we make any changes, we can use node --watch "path/to/script.js".
  • To allow a script to accept a command line argument, we can use the process module - specifically, process.argv - and get an array with any provided arguments from the command line.
  • To learn all that we can do using Node, and learn what browser functionality is and isn't available and vice versa, investigate more in the Node.js documentation.

npm

  • Node is built on a package system that uses two commands: npm and npx.
  • To install a node package, we use npm install. For example, to install Sass, we can use npm install sass.
  • Another example would be axios - we can install axios with npm install axios.
  • Each time we install a package with npm, it automatically updates our package.json and package-lock.json with information about what we installed and what version we are on.
  • To install what's in package.json, we can run npm install or npm i.
  • To initialize a new Node application or project using npm packages, we can run npm init, or npm init -y to skip the prompts and accept all the defaults.
  • Node uses what's known as the CommonJS format for JavaScript modules - you can use the require("module") command to import a file, and can save the result inside of a variable to reference and use that package. For example: 
    const fs = require("fs");

FS

  • The built-in method for working with the filesystem is bundled under a node package called fs.
  • fs has many, many methods that allow you to interact with the filesystem in a way that's reminiscent of a cli. Things like fs.writeFileSync() will write data to a file synchronously.
