CSS Positioning & Float

Position

  • By default, most elements have position: static;.
  • position: relative; positions an element relative to its default static position, using the top, left, bottom, and right properties.
  • position: absolute pulls an element out of the flow of the document, positioning relative to the nearest positioned parent - if no parent has a position of relative, absolute, or fixed, the element will be positioned relative to the document body.
  • position: fixed will position an element relative to the viewport.
  • fixed div position: fixed; bottom: 0; right: 0; width: 40rem; height: 5rem; z-index: 1;
  • position: sticky is a little different, and will stick an element based on the viewport and scroll properties. Kind of like a combination of fixed and absolute.
  • z-index: integer can be used for positioned elements only and allows you to define the absolute stacking order of positioned elements. Higher z-index will overlay lower z-index. You can't change the z-index of static content, which defaults to a z-index equivalent of 0.

Float

  • Float should not be used for layout of block elements - it's primarily to allow certain content to flow around other "fixed" content, like an image in a magazine or a newspaper.
  • Floated elements should be siblings with text-based elements they can interact with.
  • The clearfix hack can be applied to a floated element to ensure floating elements are accounted for when the parent container is figuring out its size. .clearfix:after { content: ""; display: table; clear: both; }
