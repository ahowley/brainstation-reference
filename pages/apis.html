APIs

A web API (Application Programming Interface) allows us to use HTTP requests (like GET, PUT, POST, or DELETE) to interact with data somewhere else on the web. This is the way we can expose an application to the rest of the web, or request data from another server to show and update dynamically based on user actions.

Overview

  • APIs provide a set of methods to define how we can interact with another piece of software.
  • The DOM is an example of an API.
  • An API might be protected using an API key. You can often get these by creating a developer account, then will usually pass your API key via the URL or body of your request. This will look something like: 
    "https://api.example.com/endpoint?key=CrAzYaPiKeYtExTgOeShErE"
  • Major sites often have APIs available to developers outside their company to access data from their apps.
  • A POST request sends data to another program, usually asking it to save or display the data somehow.
  • A PUT request typically would be used to update an object that already exists.
  • A DELETE request is the most self-explanatory: it requests an object or piece of data be deleted.
  • An endpoint is the URL or location you need to send an API request/"call".

JSON

  • JSON (JavaScript Object Notation) is a standardized way of passing data around the web. It was created with reference to JavaScript syntax, but isn't identical.
  • A JSON key has to be in double quotes, unlike JavaScript, and we can only include certain data types as values.
  • JavaScript has a built-in global object, JSON. This includes several helpful methods for interacting with JSON.
  • JSON.stringify(obj) will turn obj into a JSON string representation of the objects keys and values.
  • JSON.parse(jsonString) will turn a valid JSON string back into a JavaScript object that can be immediately used.

Axios

  • Axios is a promise-based HTTP Client for node.js and browsers that allows a similar interface for making HTTP requests on both the client- and server-side.
  • We can import Axios to our front-end page either in the HTML: 
    <script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js">
    Or in a script: 
    const axios = require('axios/dist/browser/axios.cjs');
  • Axios supports JavaScript's Promise API.
  • There are standard methods for each request type, like axios.get(), axios.put(), axios.put(), and axios.delete().
  • When making an Axios request, there're a few things to remember:
    • All requests should start with the await keyword, because they are asyncronous calls. This will pause execution until the call is complete.
    • If you create a function that has the await keyword somewhere inside of it, you need to define that function asynchronously as well using the async keyword.
    • Best practice is to always wrap any Axios HTTP request in a try {} catch (e) {} block. This would look something like this: 
      try{
  const response = await axios.get('url');
} catch (error) {
  console.log(error);
}
