HTTP

HTTP (Hypertext Transfer Protocol) is the foundation of the internet. This protocol is the way that data is exchanged between locations on the web.

The Basics

  • The Internet is, literally, an interconnected network built on packets of data that get passed along literal cables that carry electricity.
  • TCP stands for Transmission Control Protocol, and IP stands for Internet Protocol.
  • HTTP is the protocol that servers (or computers that hold and serve data) use to request data and respond to requests.
  • Other important protocols include SMTP/POP, which regulates how emails are sent and received, and FTP, which regulates file sharing.

HTTP Methods

  • HTTP requests include a method when they're sent. This represents the types of possible actions that can be performed on a server or data on a server.
  • GET and POST are the two most common ways we send messages to a server - GET is just like typing in a URL in a browser, and requests data. POST asks the server to save or change its data somehow.

HTTP Responses

  • When we make an HTTP request, the server will respond with a response.
  • There are many codes - you can see them on MDN here - but there are a few reliable categories.
  • 2xx response codes generally represent a success.
  • 3xx response codes represent a redirection.
  • 4xx response codes represent a client error, or the fact that the requester (client) did something wrong.
  • 5xx response codes represent a server error, or the fact that the server had something go wrong.

Web Addressses

  • A URL (Uniform Resource Locator) is structured as follows: [protocol][subdomain][domain][port][path][parameters][fragment].
  • The parameters of a URL are included with a get request, and allow data to be passed to a website so the website can respond in different ways to different requests.
  • Parameters are structured in key-value pairs: [url]?key=value&key2=value2&key3=value3

    • Axios

    • Axios is a promise-based HTTP Client for node.js and browsers that allows a similar interface for making HTTP requests on both the client- and server-side.
    • We can import Axios to our front-end page either in the HTML: 
      <script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js">
      Or in a script: 
      const axios = require('axios/dist/browser/axios.cjs');
    • Axios supports JavaScript's Promise API.
    • There are standard methods for each request type, like axios.get(), axios.put(), axios.put(), and axios.delete().
    • When making an Axios request, there're a few things to remember:
      • All requests should start with the await keyword, because they are asyncronous calls. This will pause execution until the call is complete.
      • If you create a function that has the await keyword somewhere inside of it, you need to define that function asynchronously as well using the async keyword.
      • Best practice is to always wrap any Axios HTTP request in a try {} catch (e) {} block. This would look something like this: 
        try{
  const response = await axios.get('url');
} catch (error) {
  console.log(error);
