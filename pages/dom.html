Document Object Model

The Document Object Model, or DOM, is JavaScript's way of understanding and working with HTML in real time. Manipulating the DOM via its API allows us to create webpages that change and update in real-time. For more information about anything and everything DOM, you can investigate on MDN here.

  • The DOM gives us an Application Programming Interface, or an API, to manipulate HTML elements on our webpage.
  • We can reach the entire page in JS using the built-in object called document.
  • The document object contains many built-in properties and methods that contain or change information about the page.
  • The DOM contains a list of elements on the page inside what's known as nodes.
  • We can adjust the HTML that structures the webpage in real time - one example given was directly modifying document.body.innerHTML, albeit that's a pretty unlikely use case.

Accessing the DOM

  • The reason why it's common to see a script tag at the bottom of the document is because a script can't manipulate the DOM before the DOM exists. We can also use the defer or async properties on a script tag.
  • It's unusual to use innerHTML for a number of reasons, including security, and we instead typically want to target things by element.
  • A couple of ways to access an element include document.getElementById('id'); document.querySelector('#id') or document.querySelectorAll('.class')
  • getElementById can target individual, unique elements via their assigned ID property.
  • querySelector('.css-selector') returns the first element on the page that matches the provided querySelector.
  • querySelectorAll('.css-selector') returns all of the elements on the page that match the provided querySelector as a NodeList (which is very similar to an Array).
  • It's worth noting that the return from both querySelector methods will always be empty if a pseudo-element is included.
  • It's also worth noting that all of the above methods of seleceting document objects also work when called from an element, as opposed to from document. When called from an element, they will only be able to select descendant elements.
  • Once we have an element, we can access many of its properties via its built-in methods and JS properties. For example, we can adjust in-line styles using element.style and the properties inside it.
  • To adjust an element's styles in real time, another (arguably better) option is to add or remove modifying classes to an element's element.classList.
  • To adjust an element's display text, we can work safely with an element's elem.innerText property in most cases. We can also append text to an element using element.append('text') (this does not work with appendChild())

Working with Classes

  • We can access and manipulate an elements classes with its classList, which contains a few safe and straightforward methods that allow adjusting things on the fly.
  • elem.classList.add('added-class') adds a class to the list of classes.
  • elem.classList.remove('removed-class') removes a class from the list of classes, if present.
  • elem.classList.contains('class') will tell us whether a given class is in the list.
  • elem.classList.toggle('class--modifier') is a convenient and quick way to toggle a class onto and back off of an element.

Creating Elements

  • We can create a new element from scratch using document.createElement('elem-type').
  • An element that is created this way doesn't get put on the page, but it does exist in memory. We can modify its properties as much as we want, even before it actually gets added to the document body.
  • We can set an element's HTML Attributes using elem.setAttribute('attr', 'value'). We can get an attribute's value using elem.getAttribute('attr').
  • We can add an element to the actual document by appending it to another element as a child. We do this like this: elem.appendChild(newElem);
  • There is also element.append(newElem) - these are super similar, and I'd go to MDN to find out more, cuz that site will explain it better than this one could.
  • element.insertBefore(elementInserting, childAfter) can be used to target a specific location amongst siblings, and is a helpful alternative to elem.append()
  • We can also work with nodeLists using several helpful methods, including elem.nodeList.forEach(callback).
  • We can remove an element from its parent using elem.parentNode.removeChild(elem);
