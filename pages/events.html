JavaScript Events

Events are the backbone of dynamic front-end programming - we can respond to user actions, inputs, and more by listening for events. Inspect this page to see an example mousemove event that's making the ball follow your mouse!

  • An example of events in action would be your average software's GUI. A GUI constantly listens for events the user can trigger, like clicking, keyboard buttons, and more.
  • Browser events can be handled by using one of three methods (using click as an example): we can use the onclick html parameter to assign a listener, set the elem.onclick parameter to assign a listener, or use an element's built-in addEventListener method to assign a handler.
  • When we assign an event handler inline, we can write plain JavaScript, but in general, and for both other methods, we need to write a callback function.
  • Event handler callback functions should always accept a parameter to accept the event object as an argument. event object contains a lot of useful information to help us respond properly to the event that triggered its creation.
  • If we want to handle events globally (for example tracking mouse position on mousemove), we can attach handlers directly to window: window.addEventListener('mousemove', (event) => console.log(event))
  • Events propograte upward from their target. This is known as bubbling, and can be prevented with const handler = (event) => event.stopPropagation()
  • Forms have a submit event that, by default, will submit the form data via the request type set on the form and reload the page. In order to prevent this, we can use submitEvent.preventDefault();
