DNS

DNS (the Domain Name System) allows us to host a website at a specific domain name, instead of needing to use an IP address directly. It lets us link together an arbitrary domain, with a registered top-level domain, to the IP address of our website.

  • Top-level domains include the classics like ".com", ".net", and ".org", and they are registerable by anyone (technically).
  • From there, the chain of "responsibility" flows first by any "dots" that are chained together, then to the app's routing itself. For example - "example.project.com/test" would first find ".com", which would point to "project", which would point to "example", which would point to "test".
  • DNS servers, or computers somewhere out there on the web that control DNS routing and registration, cache lookup values to accelerate response times.
  • We can set a TTL, or Time-to-Live,, to extend our DNS cache time.
  • DNS is regulated by a non-profit organization called ICANN (Internet Corporation for Assigned Names and Numbers), and the IANA oversees IP allocation.
  • You can buy (or, more aptly, lease) a domain name from a registrar like NameSilo, namecheap, AWS Route 53, etc.
  • Once you have a domain, you can use a DNS record to point that domain to the IP address of the server hosting your website.
  • Once you have a dedicated location hosting your site, the hosting service will typically show you what to enter to configure your DNS record.
  • For debugging a DNS record, whatsmydns.com, or the domain propagation tool, is useful for confirming info about your domain.
