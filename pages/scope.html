Scope

Scope in programming is a term that refers to what names and values can be referenced, returned, named, and overwritten in a given chunk of code. A common associated term is namespace - the "space" in the code within which a given name can be referenced.

"let" and "const"

  • Introduced in ECMA2015 along with arrow functions. These keywords were introduced to mimic the var keyword (the "old" way of declaring variables), except they use block scoping rules.
  • const is a special type of "variable" definition - it's not actually a "variable" at all. Instead, this keyword defines a "constant" variable - one that can never be reassigned.
  • Variables defined using let or const are scoped to the code block where they are declared. This means that cannot be referenced outside of the code where they are declared, and can re-use variable names that have already been declared outside their code block without overwriting those variables.
  • Defining variables globally (outside any code block in a script file, or more precisely, declaring a name straight to the outermost scope, "window") should be done sparingly, if ever. We want to try to make sure we declare any given name in the most specific possible scope.

"var"

  • Prior to ECMA2015, all variables were declared using the keyword var. This works just like the let keyword, except the scoping rules are different.
  • Variables defined with var are function-scoped. If you redefine a name inside a nested code block/function/scope, it will overwrite a variable declared using var of the same name from outside that scope.
  • Variables defined using var can be re-declared, while neither let nor const allow this.
  • For this reason, using let and const - i.e. the "stricter" scoping rules - is basically always preferred.

Differences Overview

  • var is the most permissive, is function-scoped, and allows both redeclaration and reassignment.
  • let is the go-to for naming variable values, is block-scoped, and allows reassignment, but not redeclaration.
  • const is the most strict, is block-scoped, and allows neither redeclaration nor reassignment.
