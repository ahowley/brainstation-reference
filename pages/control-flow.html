Control Flow

Control flow allows us to reuse, loop, or conditionally execute certain expressions, statements, and blocks of code based on the needs of a program at any given moment. We can use things like conditional statements and loops to tell a program how to behave or how many times it should execute a code block. This lets us add more structure to the instructions we write and create more powerful programs.

Conditional Execution

  • Using Conditional Statements allow us to create different branches for our code to follow depending on whether certain conditions hold true or false.
  • We create an if-else statement as follows:
    • if (someCondition === true) {
  /* someCondition is true */
} else {
  /* someCondition is false */
}
  • Not every if needs an else, it will work fine on its own.
  • else and if can also collaborate/be chained together:
    • if (someCondition === 1) {
  /* someCondition is 1 */
} else if (someCondition === 2) {
  /* someCondition is 2 */
} else {
  /* someCondition is neither 1 nor 2 */
}
  • Inside the parentheses of a conditional statement, we include a conditional expression, which will always evaluate to a boolean true or false.
  • The curly braces of a conditional statement are blocks of code - that is, stand-alone chunks of code that, in a sense, need "permission" or to be "requested" to execute.
  • Similar to if-else, but with a different syntax and common use-case, there are switch statements.
  • We execute different expressions depending on the value of the expression inside the switch statement. By default, whenever a condition is met in one of the cases, ALL remaining cases will be evaluated as "true". To avoid this, we include break; at the end of each condition. We can also provide a default code block to execute if none of the conditions are met. 
    • function getReward(tier) {
  let reward = [];
  switch (tier) {
    case "gold":
      reward.push("a gold medal");
    case "silver":
      reward.push("a high five");
    case "bronze":
      reward.push("$100");
      break;
    default:
      reward.push("a slap on the wrist");
  return reward;
  }
}
getReward("gold"); // ["a gold medal", "a high five", "$100"];
getReward("bronze"); // ["$100"];
getReward("cerulean"); // ["a slap on the wrist"];

Loops

  • Loops are a powerful type of control flow that allow us to execute the same block of code as many times in a row as we'd like.
  • One of the most common types of loop is a for loop - these are a type of loop where we (generally speaking) tell the code block exactly how many times we want it to run, and in which order.
  • A for loop is written as follows: for (let i=0; i<100; i++) { /* do stuff 100x */ }
  • The above example first sets a variable called i (a common convention, standing for "iterator") to 0. It then states that the following code block should execute as long as i is still less than 100. The final bit tells i to iterate by 1 every time the code block executes, so it gets 1 closer to the preset limit of 100 each loop.
  • There are also a few different ways to use a for loop to iterate through each element of an array. Each of the following do the same thing in different ways: 
    • for (let i=0; i<array.length; i++) {
  console.log(array[i]);
}

array.forEach((element) => {
  console.log(element);
});

for (let i in array) {
  console.log(array[i]);
}

for (let element of array) {
  console.log(element);
}
  • In addition to for loops, we also can use while loops. These execute code an unlimited number of times, so long as their condition is still true.
  • The syntax of while loops is very simple: while (someCondition) { /* someCondition is true */ }
  • We will almost always alter the condition in some way inside the looping code block, such that the condition allowing the block to loop will eventually become false.
  • The do { /* do stuff */ } while (someCondition) loop is very similar to a regular while loop, but guarantees execution of the code block at least once.
  • To break out of any loop's code block at any given moment, we use break.
