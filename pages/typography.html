Typography

See Google Fonts for a way to navigate typefaces commonly used on the web. See this page to find a snippet demonstrating the deepest possible browser support for @font-face in CSS.

Overview

  • In the context of design, typography is a tool to communicate brand and personality.
  • Typesetting is the process of taking a design and ensuring it displays property in a given domain/medium.
  • Serif fonts have little doohickeys ("a small shape or projection") at the end of lines and strokes. Like this, for example.
  • Sans Serif fonts like the one used on this website (Montserrat) don't have the doohickeys.
  • Monospaced fonts like this one guarantee every character has exactly the same width - commonly used for code.
  • Handwriting fonts are design in the style of... handwriting.
  • Display fonts are a general "other" category, including all sorts of wacky stuff.
  • A typeface is a collection of letters and symbols designed with shared patterns - a typeface also typically includes a bold, italic, thin, etc. version of the font.

In CSS

  • You can use absolute units (px) or relative units (em, rem, %) to adjust font size - you can and should choose between these depending on the responsive needs of a page.
  • Knowing when to use "px" vs. "em" vs. "rem" mostly boils down to what you want the font size to be relative to, if at all.
  • For the purposes of using "em" with margin and padding, this is not always relative to the parent. Instead, it looks for the nearest defined font-size in the ancestry tree, including itself.
  • font-weight either accepts a numeric value (100, 200, 300, etc.) or a pre-defined keyword value (bold, thin, etc.) as defined in the typeface to change the thinness/thickness of a font.
  • font-style allows you to change the style among typeface options (normal, italic, oblique).
  • letter-spacing adjusts the spacing between letters. Typically shouldn't be used for large amounts of text, but instead for specific headings or titles. This is equivalent to tracking in typography, not kerning
  • line-height is the equivalent of leading in typography. It adjusts the spacing between each line of text. Can be set with numbers, without units! For example: line-height: 2 represents a total line-height equal to 2 times the font size property of the element we're styling.
  • @font-face is an at rule used to load a custom font-family into a website. Can be loaded from a remote location or from a directory in the project. Can also pull from the local user's computer, technically. The two mandatory properties are font-family (the name of the font as referenced in code) and src (the path to the resource used to define the font)
  • Define @font-faces using standard CSS properties such as font-weight and font-style, and define multiple font-faces for one font to define which font to apply to various CSS keyword values, like bold and italic.
  • At rules can be used to declare resuable CSS, or to apply conditions to blocks of CSS. Some common ones include: @media @font-face @keyframes @charset @import @namespace
  • ".woff2" font files were used in the examples given during the BrainStation lecture, and is supported by most browsers.
  • Specifying format in the src line is best practice: src: url("path/to/font.woff2") format("woff2")
  • Convention also says to put font definitions in their own CSS or SCSS file.
  • Multiple @font-face definitions for the same typeface should have the same font-family name.
  • The format() for a ".ttf" is "truetype"., and for ".otf" it's "opentype".
  • To add support to other font formats, like ".woff" and ".ttf", you can add multiple sources in the src property, separated by a comma.
