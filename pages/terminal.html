Terminal Quick Reference

  • A terminal is the application that runs commands on a computer.
  • A command line is the user interface where you type in commands for the terminal to run.
  • The shell is the program or interpreter that processes user input.
  • Standard Output or stdout is where the computer remembers output from commands - the command line reads from this to display the lines you see.
  • Unix is a long-standing terminal-based operating system, out of which grew most modern terminals. Terminals that mimic Unix's structure/syntax are known as Unix-Like.
  • Commands are structured with flags and arguments. command -flag "argument"
  • A directory is, essentially, the same thing as a folder. Maybe more accurate is to say a folder is another way to represent a directory. It's a place where files (and other directories) can be stored. Directories are located in a filesystem stored on a drive.

Commands

  • $ echo "text" takes the text input provided and spits it out in response.
  • tab will attempt to auto-complete whatever you're typing.
  • ctrl + c will exit most currently-executing commands, tasks, or programs.
  • up/down arrows let you navigate through your command history or re-run your previous commands.
  • man command shows the man(ual) page for a given command, including all documentation for the command's functions. This won't work in Git Bash on Windows, but you can often use command --help instead.
  • clear will clear the current output from the screen. This doesn't clear stdout - it actually writes to it in order to clear the screen.
  • pwd, or "Present Working Directory", will output the current working directory.
  • mkdir "path" will create a new directory/folder located at "path".
  • ls will list all files and folders in your current working directory. ls -a will include all hidden files and folders. ls -l lists entires using a "long listing" format.
  • cd "path/to/directory" ("Change Directory") allows you to change your current working directory. Enter the directory you'd wish to travel to as an argument using path formats you're already familiar with from html, including cd .. to travel into the parent directory.
  • touch "path/to/newfile" lets you create a new file in a given path.
  • cat stands for "Concatenate" and is a flexible command that lets you handle text data via commands. A simple usage is cat file, which will print file contents to stdout.
  • > and >> are special commands that capture output from other commands and write or append that output into a file. For example: cat file_1.txt > file_2.txt will overwrite the contents of file_2.txt with the contents of file_1.txt, and cat file_1.txt >> file_2.txt will copy the contents of file_1.txt into the end of file_2.txt without overwriting what's already there.
  • mv "path" "new/path" lets you move a file or folder to a new directory.
  • cp "path" "path/to/copy" lets you copy a file or folder.
  • rm "path" lets you delete a file. To recursively force delete a folder, use rm -rf "path".
  • code "path/to/folder" to open "folder" in VSCode.
